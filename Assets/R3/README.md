# R3 Evidence Form

Digital R3 evidence form for SPEA / governance observations at AIS Sharjah. Same architecture as the lesson observation form: HTML submits to an Apps Script web app, which appends a row to a Google Sheet, and emails a backup copy with a per-record locked-view URL to `admin.user@ais.ae`.

From v0.29 the live form is **password-gated with StatiCrypt** and locked records require a per-record token (no more ID-guessing).

## Edit / deploy workflow

1. Edit `src/r3-evidence-form.html` (the master).
2. Run `./encrypt.sh` from this folder. It produces a fresh `r3-evidence-form.html` (the encrypted output).
3. Commit both files, bump version footers, tag, push (`git tag v0.X && git push origin main --tags`).
4. Force a Pages rebuild: `gh api -X POST repos/Rogerceaser21/Data-Representation/pages/builds`.

To rotate the school-wide password: edit `PASSWORD=` in `encrypt.sh`, re-encrypt, commit.

## Apps Script + Sheet

Backend is the `apps-script/` clasp project (running as `admin.user@ais.ae`). Submissions:

- Append a row to the `Submissions` tab (48 columns; last is `record_token`).
- Generate a 32-char hex token, stored alongside the record.
- Email `admin.user@ais.ae` an HTML summary + the locked-view URL (token included).

Gmail filter to auto-archive + label submission emails (set up once in Gmail UI):

> From: `admin.user@ais.ae`  
> Subject contains: `AIS R3 Evidence ·`  
> → Skip Inbox, Apply label `R3 Submissions`.

After the first deploy with the new `mail.send` scope, open the script in the browser editor and run `forceAuth` once to grant the email permission.
