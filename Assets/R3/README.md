# R3 Evidence Form

Digital R3 evidence form for SPEA / governance observations at AIS Sharjah. Same architecture as the lesson observation form: HTML submits to an Apps Script web app, which appends a row to a Google Sheet, and emails an HTML backup copy with a per-record locked-view URL to `admin.user@ais.ae` and the selected inspector.

Currently at **v0.37**. From v0.29 the live form is **password-gated with StatiCrypt** and locked records require a per-record token. From v0.34 Tom Select uses **opaque indices** as option values to sidestep an iOS WebKit bug that was blanking dropdowns on iPad Safari + iPad Chrome. From v0.37 the options fetch is self-healing (silent timeout + retry + stale-tab reload) so the loading spinner can never hang for a stakeholder.

## Edit / deploy workflow

1. Edit `src/r3-evidence-form.html` (the master).
2. Bump the version footer inside the master (and in the Lesson Observation form, to keep the project-wide stamp in sync).
3. Run `./encrypt.sh` from this folder. It produces a fresh `r3-evidence-form.html` (the encrypted output).
4. Commit, tag, push: `git add ... && git commit -m "vX.Y · ..." && git tag vX.Y && git push origin main --tags`.
5. Force a Pages rebuild: `gh api -X POST repos/Rogerceaser21/Data-Representation/pages/builds`.

To rotate the school-wide password: edit `PASSWORD=` in `encrypt.sh`, re-encrypt, commit.

## Apps Script + Sheet

Backend is the `apps-script/` clasp project (running as `admin.user@ais.ae`). On submit:

- Append a row to the `Submissions` tab (48 columns; last is `record_token`).
- Generate a 32-char hex token, stored alongside the record.
- Compute `duration` server-side as integer minutes from `time_in` and `time_out` (v0.32).
- Email `admin.user@ais.ae` an HTML summary + the locked-view URL with token; CC the selected inspector if their `Inspectors!B` email cell is populated (v0.31).

To redeploy after a backend edit (URL stays stable):

```bash
cd Assets/R3/apps-script

```

## Cache invalidation (v0.33)

`02_doGet.gs` `getDropdownOptions()` is wrapped in `CacheService` with a 5-minute TTL. If you edit the Teachers / Inspectors / Curriculum / Subjects tabs and need the form to see the change before the TTL expires, run `clearOptionsCache` from the Apps Script editor's function dropdown. Otherwise the form serves the cached options until natural expiry.

## Gmail filter (one-time)

Set up once in Gmail UI on the `admin.user@ais.ae` mailbox so submission emails auto-archive into a labelled folder:

> From: `admin.user@ais.ae`  
> Subject contains: `AIS R3 Evidence ·`  
> → Skip Inbox, Apply label `R3 Submissions`.

After the first deploy with the `script.send_mail` scope, open the script in the browser editor and run `forceAuth` once to grant the email permission.
