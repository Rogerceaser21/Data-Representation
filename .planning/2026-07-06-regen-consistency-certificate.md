# Regeneration Consistency Certificate · R3 June 26 benchmark

**Date:** 2026-07-06 · **Task:** Focus OS `ea033575` (verify next-round AI prompts + sandbox)
**Question answered:** if the future "round finished" button regenerates every AI-written dashboard
text through the standalone Anthropic API path, how consistent is the output with the approved June
publication, and can fabricated content ever ship?

## Method

- The approved June output (57 narrative claims, 652 references, 104 coaching notes) and the June
  input data were FROZEN with sha256 manifests (`~/AIS-Data-Dashboard/bench/june-2026/`).
- Four standalone Node pipelines (the button's exact call path: Anthropic API, enforced JSON
  schemas, streaming, prompt caching) regenerate the text from the frozen input only. Live
  Supabase is never touched; candidates are files, viewable in a local sandbox dashboard.
- An offline judge scores every run: LAYER 1 hard gates (verbatim quotes from a single inspector's
  text, own-teacher evidence only, rubric verbatim, curated links only, anonymity, exact structure)
  and LAYER 2 consistency vs the benchmark (numbers, evidence grounding recall, structure, meaning-
  scored wording). Calibration: the benchmark judged against itself scores exactly 100% and six
  deliberately sabotaged copies (fabricated quote, changed number, dropped claims, fake link,
  teacher named, invented evidence) are all caught.

## Certified recipe (the "winner" of the model/prompt tournament)

| Step | Choice |
|---|---|
| Generation | **Claude Sonnet 5** + the approved previous narrative as EXEMPLAR (adaptive thinking, default effort) |
| Reference matching | **Claude Sonnet 5** (exhaustive per-claim matching; Haiku loses ~3 pts on quote precision) |
| Meaning checks | Claude Haiku 4.5, temperature 0 |
| Deterministic guards | verbatim-cleaners (chunk-scoped), per-lesson dedupe, insight-count reconcile, TRUE_NUMBERS validation, evidence lines generated by CODE not AI |
| Acceptance | hard gates must pass; composite ≥95% or auto-retry |

**Anti-fabrication: 100% in every run.** Not one non-verbatim quote, invented count, fake link or
teacher name survived to output; the cleaners caught and dropped the (few) near-verbatim slips.

## Key findings

1. **The exemplar is the consistency engine.** Without it: ~50% meaning consistency (the model
   picks different true findings). With it: 88-90% stable across reps.
2. **The benchmark's own numbers went stale** (late June forms moved 47→49 lessons after the text
   was approved), so number-consistency is certified as equality-OR-truth (verified against the
   claim's own evidence count or the computed statistics). Evidence lines become code-generated.
3. **The AI cites more evidence than the human benchmark** (≈20 vs ≈12 refs/claim at 72-79% recall
   of the human citations); recall is the certified grounding metric.
4. **Same config varies run to run** (~3-5 composite points, occasional gate-catchable slips):
   the button MUST loop generate → judge → reconcile → retry. With that loop, the sub-95 rep
   class is recoverable; certified band with one retry: ≥95% on P1+P2, 85-90% on P3/P4 meaning.
5. **Operational:** enable API auto-reload before round day (a mid-run credit wall killed one run);
   full-stack regeneration costs ≈$8-12/round at intro pricing (halved by caching, already wired).
