# Next-round AI regeneration prompts

> Reusable, round-parameterised prompts that recreate every AI-written dashboard text surface for a NEW R3 round. Stored in THREE places (this MD, Supabase `ai_prompts` via `get_ai_prompts()`, and one Focus OS task as JSON) for redundancy. Canonical source: `~/AIS-Data-Dashboard/db/ai_prompts_build.mjs` (re-run it to update all three). `${ROUND}` = the new round label.

> **Certified 2026-07-06** (recipe entry `recipe`, ord 0): models, exemplar mechanism, deterministic guards, judge and >=95% acceptance loop are verified end-to-end. Certificate: `.planning/2026-07-06-regen-consistency-certificate.md`. Read the `recipe` entry before running any pipeline.

## Global rules (apply to every pipeline)

- Round parameter: replace the round label everywhere (tools take it as argv or a ROUND const; app_config.current_round is the live value).
- Best evidence wins: an overall is the MIN score (1 = best), never a mean, when rolling up observations.
- Dedupe to lessons: collapse co-observed forms by teacher + date + normalised subject area, best score wins, evidence text merged. Counts are LESSONS, not raw form rows.
- Every quote used as evidence/reference MUST be a verbatim substring of the source observation text. No paraphrase. An automated pass drops any quote that does not match.
- References NEVER name a teacher. Never link one teacher to another teacher’s R3 record (anonymity breach).
- User-facing label is always "Insights", never "AI".
- Observer shorthand: SS = Student, T = Teacher, S to S = Student-to-Student dialogue.
- Australian English. No em dashes or en dashes anywhere (use commas, parentheses, semicolons). Hyphens in compound words are fine.
- phase = primary_kindy (teachers.section in kindy/primary) or secondary; plus a compare view.


## 0. Certified regeneration recipe (verified 2026-07-06): read before running any pipeline  `(recipe)`

**Produces:** the verified configuration every pipeline below runs under (models, guards, acceptance)  
**Inputs:** The previous approved round as EXEMPLAR + the frozen benchmark/judge tooling in ~/AIS-Data-Dashboard/bench (STATUS.md, bench_judge.mjs, regen_*.mjs).

**Pipeline:**
- GENERATE: run the pipeline on Claude Sonnet 5 with the previous approved round passed in as an EXEMPLAR (the consistency engine: ~50% meaning consistency without it, 88-90% with). Enforce the JSON output schema (Anthropic output_config.format); stream when max_tokens > 16k.
- CLEAN (deterministic, in code, never trusted to the model): chunk-scoped verbatim-quote cleaner (a quote must be an exact substring of ONE inspector text chunk; quotes spanning the "||" co-observer merge are dropped), per-lesson dedupe, insight-count reconcile (refs-count semantics only), TRUE_NUMBERS validation (band counts/totals computed from the data), and evidence lines ("Drawn from ... X of Y lessons.") generated by CODE.
- JUDGE (offline, bench_judge.mjs): LAYER 1 hard gates (verbatim quotes, own-teacher evidence, rubric verbatim, curated links, anonymity, exact structure) + LAYER 2 consistency (numbers = equality-OR-truth, grounding = RECALL of prior refs, structure, meaning via Haiku temp 0). Self-calibrates at exactly 100% on the benchmark; a 6-way sabotage suite is all caught.
- ACCEPT or RETRY: hard gates must pass (zero fabrication) AND composite >= 95 (P1+P2) / meaning 85-90 (P3/P4); else loop generate -> clean -> judge -> retry. Run-to-run variance is ~3-5 composite points, so the loop is mandatory; nothing lands in Supabase until it passes.

**Prompt:**

> Before running any pipeline below, load the certified recipe. Every AI-written surface is regenerated on Claude Sonnet 5 with the previous approved round as an exemplar, forced through a JSON schema, cleaned by deterministic code guards (verbatim-only quotes, per-lesson dedupe, computed TRUE_NUMBERS, code-generated evidence lines), then judged offline against hard anti-fabrication gates and a >=95% consistency bar with an automatic retry loop. No output that fails a gate or the bar may reach Supabase or the dashboard. Two delivery engines share these exact prompts, judge and seeding: Route A = the Anthropic API key, headless, driven by a Supabase job-queue row; Route B = Igor’s Mac running Claude Code on the Max plan (a "claude -p" poller on the same queue, no API dollars), pinned to the SAME Claude Sonnet 5 so it stays inside the certified band. Model (Sonnet 5) and engine are separate axes: the pipelines take an --engine api|claude-code flag, but BOTH engines run Sonnet 5.

**Rules / locked decisions:**
- CERTIFIED, do not change without re-certifying: generation + reference matching = Claude Sonnet 5; meaning checks = Claude Haiku 4.5 (temperature 0). BOTH delivery engines run this same Sonnet 5: Route A via the API key, Route B via "claude -p" pinned to Sonnet 5 (NOT Claude Code’s default model). Switching models voids the certificate; Fable via API would ~5x the per-round cost for no gain.
- The EXEMPLAR (previous approved round) is mandatory for narrative + recommendations; it is what makes the output reproducible (without it ~50% meaning consistency, with it 88-90%).
- Anti-fabrication is enforced by CODE gates, never trusted to the model: verbatim-substring quotes only, own-teacher evidence only, curated links only, never name a teacher, exact cell structure. All runs in certification were 100% gate-clean.
- Numbers metric = equality-OR-truth (benchmark text goes stale as late forms arrive; truth = the claim’s own verified ref count or a computed statistic). Grounding metric = RECALL of the prior round’s references (the AI cites MORE than the human benchmark, ~20 vs ~12 refs/claim).
- Acceptance = gates pass AND composite >= 95 (narrative/refs) / meaning 85-90 (recs/coaching), else auto-retry. The button MUST loop; a single pass is not certified.
