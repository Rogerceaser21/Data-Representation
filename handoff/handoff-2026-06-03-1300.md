### The Tom Select integration

- Library vendored locally at `Assets/R3/lib/tom-select.css` + `tom-select.complete.min.js` (v2.6.1, Apache 2.0)
- Loaded synchronously in `<head>`
- `baseOpts` includes: `create: false`, `maxItems: 1`, `maxOptions: 1000`, `plugins: ['clear_button']`, `dropdownParent: 'body'` (body-attached so `.info-grid { overflow: hidden }` can't clip), `closeAfterSelect: true`
- `numberOpts` extends baseOpts with `sortField: null`, `controlInput: null` (no typing), `dropdownClass: 'ts-dropdown number-picker'` (triggers grid CSS)
- 7 Tom Select instances managed via `tomSelects` object: teacher, inspector, subject, grade_class, num_sen, num_gt, num_male, num_female, num_on_roll, present

### School-aware Subject filtering

`refreshSubjectOptions()` re-populates the Subject Tom Select whenever the School hidden input changes (via change-event listener wired in init). Filter logic: `SUBJECTS_RAW.filter(s => s.active && s[schoolKey])`. Subject is disabled with "Select School first" placeholder until School is picked.

## What's next · password-protect (v0.28)

Igor said "I'll tell you the password once I think you know what and how I will want you to do this". He has confirmed (and the prior agent has researched) that he wants **StatiCrypt** password protection, same pattern as his `Tutoring Aaira` sibling project.

### Research done this session

- Tool: [StatiCrypt](https://github.com/robinmoisson/staticrypt), client-side AES-256-GCM
- Usage: `npx --yes staticrypt <file> -p '<password>' --short --template password-template.html --template-* ... -d encrypted-out`
- Custom gate UI template lives at `<deploy>/password-template.html` and is hand-crafted to match the protected document's aesthetic (peach/sage for Tutoring Aaira; would need navy/cream/green for R3)
- Reference repo: `github.com/Rogerceaser21/sample-student-report` (Tutoring Aaira deck)
- Reference deploy folder: `<root>/../Tutoring Aaira/Tutoring Aaira/HTML/deploy/` and `<root>/../Tutoring Aaira/Tutoring Aaira/HTML AIS Report/deploy/`
- Reference README: `<root>/../Tutoring Aaira/Tutoring Aaira/HTML AIS Report/README.md` (has the exact StatiCrypt command Igor uses, plus deploy workflow)

### Open architecture question for v0.28

The Tutoring Aaira pattern uses a separate `deploy/` subfolder with its own git repo pointing at the Pages-serving repo. This Data-Representation project is structured differently — the form HTML is served directly from `main` branch at `Assets/R3/r3-evidence-form.html`. Two paths:

1. **Restructure**: split master vs deploy, like Aaira. More work but mirrors the proven pattern.
2. **In-place encrypt**: overwrite `r3-evidence-form.html` with the encrypted version on push, keep the unencrypted source in `Assets/R3/src/` or similar. Simpler.

**Ask Igor before picking.**

### Gotchas to anticipate

- The form's Apps Script POST + GET work fine after decryption (StatiCrypt only protects HTML at rest; once decrypted the JS runs normally)
- Tom Select library files (`lib/tom-select.css`, `lib/tom-select.complete.min.js`) need to be accessible from the encrypted page. Either: don't encrypt them (they're library code), or include them inline in the master before encryption (bloats the encrypted blob).
- The `lesson-observation-form.html` is a separate file that Igor hasn't mentioned protecting yet — likely also needs the same treatment for consistency.
