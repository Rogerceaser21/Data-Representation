### New Sheet tab: "Observed Term 3 - Week 12" (gid 1743616402)

A live, formula-driven dashboard in the R3 Sheet (id `1V1Nb8hTWN-FpDps2Q_-F0TBWBFnPHVBW_sicWk8XVKo`). All built via the `gws sheets` CLI (no Apps Script, no git). Structure:

- 2-row grouped header (Title row; "Observation 1/2/3" merged group row; sub-header row). Data starts row 4, all 171 teachers. Frozen 3 rows + 3 cols (School/Teacher/Times stay visible).
- Per observation block: **Obs (tick) · Subject · then 10 judgement-average columns**: Att · Prgs · Skill · Innov · Teach · Assess · Curr · PSGS · Lead · Othr (these are the user's exact labels; "PSGS" is intentional for PCGS). Score columns are skinny with **vertical headers**. Helper date columns AN/AO/AP are hidden.
- **Counting = distinct observation DATES, not raw submissions** (two inspectors same teacher same day = 1 observation). Uses `COUNTUNIQUEIFS(Submissions!F, Submissions!C, teacher)`. IMPORTANT: an earlier attempt used `COUNTA(UNIQUE(FILTER(...)))` which is BUGGED — `COUNTA` counts the FILTER "no match" error as 1, ticking every teacher. Do NOT reintroduce that pattern.
- **Subject per observation** = first submission's subject on that date (option A).
- **Judgement averages**: `AVERAGEIFS(scoreCol, teacher, date, scoreCol>=1)` wrapped in `IFERROR(ROUND(...,1),"")`. The `>=1` excludes blanks AND zeros; blank = "not assessed on that point" → cell left blank, never 0/NA. 1 decimal.
- **Colour coding (conditional rules, 1=best=green … 6=worst=red)**: 1.0-2.9 green (white text), 3.0-3.9 yellow (navy text), 4.0-4.9 orange (white), 5.0-6.0 red (white). NOTE: this was SWAPPED late in the session — the user first gave it reversed, then corrected (1 is best). Current state is correct.
- Submissions col map (for any future formula work): teacher=C, inspector=E, observation_date=F, subject=L, school=AU, record_token=AV. Judgement SCORE cols: Att=X, Prgs=Z, Skill=AB, Innov=AD, Teach=AF, Assess=AH, Curr=AJ, PCGS=AL, Lead=AN, Othr=AP.
