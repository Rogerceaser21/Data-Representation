### The stuck-on-loading bug (root cause, fixed in v0.37)

A stale iPad tab left open for a day+ (suspended by iOS / in bfcache) hung on the loading spinner after the user typed the password. Root cause: the dropdown `fetch` had **no timeout**, so when a resumed-from-suspension tab fired it into a stalled JS context, it never resolved/rejected and the frosted overlay never cleared. Confirmed via research agent (web.dev bfcache docs + SvelteKit #5415 reproduction). v0.37 fixes it invisibly. **StatiCrypt and the v0.34 opaque-value fix were ruled out as causes.**
