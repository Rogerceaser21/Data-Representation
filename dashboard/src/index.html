Build history
v0.39The Snapshot gains a Report button: choose the Simple report (the current summaries + insights, references behind the side panel) or, soon, the In Depth report. Each report can be exported to a real Google Doc that mirrors the on-screen layout (clean report in the main tab, every source in a References tab, and each claim's number links to it). The insight text and its references are now read from one place (Supabase), with the page only keeping a copy for instant loading. The In Depth report is a button for now and is built next.
v0.38Settings is now password-gated and saved to Supabase, so an admin change applies on every device. Clicking Settings pops up a password prompt (the same admin password as the R3 form, checked server-side); Settings opens only after it is entered, and it re-asks every time you open it. The light or dark toggle was removed from Settings, the round button top-right is now the only theme control. The star-field on/off, the three star sliders and the Snapshot auto-play are saved to Supabase (read on load, written when you change them); the password and the settings come from one shared place, so changing the password once moves both the dashboard and the R3 form.
v0.37Story controls moved up into the tabs row. The play / pause / step / dots / Present controls are now a compact pill between the Primary & Kindy phase toggle and the Story / Detail toggle (they were a larger bar below the stage). Smaller buttons; they show only on the Story view and still float to the bottom in Present. Layout only; the controls and dashboard logic are unchanged.
v0.36Print rendering fixes (print only; the on-screen dashboard is unchanged). Background colours are now forced to print, so the charts, bars, dots, gauges and the navy cover slide are no longer blank on paper. Story slides anchor each slide's picture and figures to their own page instead of piling up at the bottom. The Coverage stat cards hug their text and wrap rather than stretching across the full page. Bars print at their final size.
v0.35Coverage corrected + a Print button. Coverage now counts the same way as the Snapshot: the June R3 round only (February dropped) and distinct lessons with co-observations merged, not raw form submissions, so the two boards agree (149 on register, 104 seen in June, 45 not yet seen, 110 lessons). A new Print / Save-as-PDF button (top right) prints the current board: the Story prints every slide one per page; the card views (Detail, Coverage, Observations) print their cards without ever cutting one across a page break, forced to light. Works from the button or Cmd / Ctrl + P.
v0.34Snapshot Story polish. The stage is taller, so the two Insights slides (5 and 7) show all three cards without the last one clipping. Present mode now opens in real full screen (not just filling the browser), every beat's text and figures are larger and centred, and an A- / A+ control in the Present bar resizes the Story text live (50 to 200 percent, session only). The Present-mode changes are scoped to Present; the embedded Story, Coverage and Observations are unchanged.
v0.33Headers made consistent across the three boards and tightened. The Snapshot subheading ("110 lessons observed…") was removed for a more compact header. Coverage and Observations now use the same two-column header as Snapshot (kicker + timestamp on the left, title right-aligned). On Observations the R3 Obs control moved to a row under the header (where the Snapshot tabs sit), the page title reads "R3 Observations", and the "Back to Coverage" button was removed.
v0.32Snapshot header restructured: the title and subheading are right-aligned beside the SNAPSHOT kicker, the timestamp moved under the kicker on the left, and the phase tabs (Primary / Secondary / Compare) now share one row with the Story / Detail toggle. In Detail, the Insights are split into two side-by-side cards (Strengths and Areas to develop) matching the What is working / Where to focus cards above them.
v0.31The sources panel now opens from the RIGHT and pushes the page aside instead of covering it, so you can read the Story and its sources at the same time. The rest of the page is only lightly dimmed (not greyed out) and its markers stay clickable, so you can jump straight from one source to another. The big Story section headers (the phase title and the progress title) are now the AIS navy and centred.
v0.30Snapshot Insights now carry their sources. Every summary, bullet and insight shows a small numbered marker; clicking it opens a panel from the left with the supporting lesson quotes (subject, date and progress word) and a link to the locked R3 record, plus a short "how this is counted" note. The wording and the references are read live from Supabase per round (the baked copy is the offline fallback). References never name a teacher. Snapshot only; Coverage and Observations untouched.
v0.29The observer illustration on the two Insights slides (5 and 7) is now half its previous size, closer to the teacher figure on the numbers slide. And the teacher figure, which used a slightly different blue, is recoloured to the approved AIS blue (#1257FF, Blue on Star) so it matches the presenter headers, which were already that blue. Story only.
v0.28Autoplay now loops: when the Story reaches the last slide it returns to the first and keeps playing. And fixed blank content after backgrounding the tab and returning (the data-missing, needs-refresh bug, the same as the R3 form): the dashboard now re-renders the current board when the tab is restored, so nothing stays blank and no manual refresh is needed. The open teacher and the story position are kept.
v0.27Fixed two Story bugs with one root cause. The freeze-safety cleanup that runs after each beat was wiping the beat’s custom inline styling, not only the animation leftovers, so the slide 4 and 6 headlines jumped from their small size to the large base size on first load, and the slide 4 to 7 headers reverted from green or terracotta to blue while playing. The cleanup now clears only the animation properties, so the custom size and colour are kept. It was never a font problem. The Observations board, which shares the same cleanup, is unaffected.
v0.26The Story view was rebuilt to the new illustrated design. The cover is a navy classroom scene, the headline numbers carry pencil motifs (book, teacher, progress), and the What is working, Where to focus and insight beats gain a margin figure, with drifting motes and a per-beat progress bar. Every number still comes live from Supabase (the deduped lessons, teachers, average and distribution) and the wording is unchanged from the Detail view. The data visuals (the dot-sort, gauge and diverging chart) stay. Coverage, Observations and the Detail view are untouched.
v0.25The Story was rebuilt so the data itself carries the animation, not decorative pictures. Each lesson is now a dot that flies in and sorts onto the six-point rating scale, the headline numbers count up, the gauge sweeps, and the cover uses a custom AIS image. It uses only the wording and figures from the Detail (v1) view. The hand-drawn graphics were removed.
v0.24The Snapshot now opens on an animated Story view, the new landing page. It plays the June picture beat by beat (cover, the teachers and lessons seen drawn as illustrations, progress on the gauge and distribution, what is working, where to focus, and the insights), one after another. It auto-plays and can be paused, stepped with the arrows, turned off completely (Settings), or shown full screen with Present. The original card view is one click away as Detail. Coverage and Observations are unchanged.
v0.23Snapshot now counts distinct LESSONS, not form submissions. Most June lessons were co-observed by two inspectors who each filed a form, so the raw rows (167) double-counted; the Snapshot now collapses submissions into lessons (teacher, date, subject) with the best score winning, giving 109 lessons (Primary and Kindy 47, Secondary 62) and four Very Weak progress lessons. Average ratings and teacher counts are unchanged. The Observations board is untouched and still shows every individual submission.
v0.22New Snapshot board, now the landing view. A governance "at a glance" picture of the June R3 sweep with a Primary and Kindy / Secondary / Compare toggle. Each phase shows lessons observed, teachers seen, average progress (best evidence per teacher) on a ramp gauge and a six-point distribution, a What is working and Where to focus next summary, and three insights for each drawn from the inspectors' written evidence. Compare puts the two phases side by side with a diverging distribution chart. Teacher subschool (section) is now populated in Supabase from the staff register, so the split is live data, not hard-coded.
v0.21Dashboard now reads LIVE from Supabase. It paints the baked snapshot first (instant + offline fallback), then upgrades to live data via a read-only Supabase RPC and re-renders, so edits in Supabase show on refresh. Login-less for now (the password gate is the soft lock); proper @ais.ae sign-in comes later. If Supabase is unreachable it silently stays on the baked snapshot.
v0.20Masthead cleaned up: the Stars and theme buttons were removed from the top bar (both still live in Settings). A floating light/dark toggle, the same round sun/moon button as the R3 form, now sits in the top-right corner and is always available.
v0.19R3 Obs is now a fan-out menu of observation levels: Teacher Level (the working teacher search), Grade / Department Level, and School Level (which expands to Kindy, Lower Primary, Upper Primary, Whole Primary, Secondary). Teacher Level works as before; the other levels land on a styled placeholder for now (front-end only, to be connected to data later). The R3 Obs button stays as the menu trigger in every state.
v0.18Fixed the search: the search bar was leaking into the hub state (a class display rule was overriding the hidden attribute), so it appeared before R3 Obs was clicked and filtered a list that was not shown. Now the hidden attribute always wins, so the hub shows only R3 Obs and the search filters once you enter it. Each tab now has its own URL too: Observations is /#teachers, Settings is /#settings, Coverage is the default; the link changes, is bookmarkable, and back/forward work.
v0.17Observations tab tightened: the R3 Obs button and the teacher search now sit inline on the "Observations" header line (whichever state is active) instead of stacked in their own panels, reclaiming the wasted space. The coloured A / B index letters were removed from the Coverage and Observations tabs (and the redundant R3 from the button), and the kicks are now just "Coverage" and "Observations".

No observation detail shows until a teacher is picked, after which only the search bar stays at the top with the selected teacher's detail below. Typing again re-shows the list to switch teacher. Back returns to Coverage.
